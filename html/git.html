Система контроля версий Git

Работа с репозиторием

  • GIT INIT - Инициализирует новый репозиторий Git в текущей директории.
  • GIT CLONE <URL> - Клонирует удаленный репозиторий на локальный компьютер.
  • GIT STATUS - Показывает состояние рабочего дерева и индекса, включая измененные, добавленные и удаленные файлы.
  • GIT LOG - Отображает историю коммитов.

Подготовка изменений

  • GIT ADD <FILE> - Добавляет указанный файл в индекс для подготовки к коммиту.
  • GIT ADD . - Добавляет все измененные и новые файлы в индекс.
  • GIT RM <FILE> - Удаляет файл из индекса и рабочего каталога.
  • GIT RESTORE <FILE> - Отменяет изменения в указанном файле, не добавленные в индекс.

Коммиты

  • GIT COMMIT -M "<MESSAGE>" - Создает новый коммит с указанным сообщением.
  • GIT COMMIT - Открывает редактор для написания сообщения коммита.
  • GIT COMMIT --AMEND - Изменяет последний коммит.
  • GIT REVERT <COMMIT> - Создает новый коммит, отменяющий изменения указанного коммита. /li>
